## What is feature()

Claude Code uses the Bun bundler's `bun:bundle` module to provide compile-time feature gating:

```typescript
// Usage in source code (src/tools.ts, etc.)
import { feature } from 'bun:bundle'

const SleepTool = feature('PROACTIVE') || feature('KAIROS')
  ? require('./tools/SleepTool/SleepTool.js').SleepTool
  : null
```

In Anthropic's internal build, `feature()` is evaluated at packaging time - code that returns `true` is retained, and code that returns `false` is completely removed by **Dead Code Elimination (DCE)**.

In our decompiled version, this function is summarized as:

```typescript
// src/entrypoints/cli.tsx line 3
const feature = (_name: string) => false;
```

This means that the code after all 88+ feature flags is never executed at runtime, but the code itself remains intact and can be read and analyzed.

## Typical patterns in code

Feature flags have three main usage modes in code:

### Mode 1: Conditional loading tool

```typescript
// src/tools.ts — most common patterns
const MonitorTool = feature('MONITOR_TOOL')
  ? require('./tools/MonitorTool/MonitorTool.js').MonitorTool
  : null
```

When flag is `false`, `require()` calls are removed by DCE and the tool does not appear in the list of available tools.

### Mode 2: Conditional registration command

```typescript
// src/commands.ts — Register slash commands
if (feature('VOICE_MODE')) {
  commands.push({ name: 'voice', description: '...' })
}
```

### Mode 3: Conditionally enable API features

```typescript
// src/constants/betas.ts — controls the beta headers sent to the API
export const AFK_MODE_BETA_HEADER = feature('TRANSCRIPT_CLASSIFIER')
  ? 'afk-mode-2026-01-31'
  : ''
```

<Note>
Since `feature()` is evaluated at build time, the code removed by DCE does not increase the final bundle size. But in the decompiled version, this code is all preserved - which is why we were able to perform a complete analysis.
</Note>

## Interesting findings

- **KAIROS family** is the largest - 6 related flags control everything from core functionality to push notifications
- **ABLATION_BASELINE** is used for "scientific controlled experiments" - it turns off advanced functions such as thinking, compaction, auto-memory, etc., and measures the baseline performance of bare API calls
- **BUDDY** is an AI mascot/elf system - fully implemented in the `src/buddy/` directory
- **ULTRAPLAN** and **ULTRATHINK** imply more advanced reasoning modes than the current extended thinking
